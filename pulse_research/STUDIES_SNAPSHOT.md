# PREREG — **BTC-beta-hedged Radar** (тест №2 семейства)

СТАТУС: **ЗАМОРОЖЕН 2026-07-10 ночью, ДО открытия данных.**
Брат отменил своё правило ждать 20.07 (оно было защитой от эмоционального
«заполнить пустоту», не научным ограничением; тест изолирован от живого проекта
и ритуала). Путь A: вето Codex учтено дословно. После этой печати — менять НИЧЕГО
нельзя, только датированный bugfix-лог §9.
Переименование (Codex): BTC-хедж убирает BTC-бету, но НЕ весь общий alt-factor —
поэтому тест называется BTC-beta-hedged, а НЕ market-neutral.

## 0. Вопрос
Не «растёт ли монета после сигнала» (закрыто: KILLED, six_year_RESULTS). А:
**перерастает ли монета с radar-сигналом рынок после хеджирования беты** —
т.е. есть ли в ОТБОРЕ радара относительная альфа, или он ловил общую альт-бету.

## 1. Форма (торговая, исполнимая)
- Лонг: доставленный Radar-кандидат — **ровно замороженный
  backtest_6y/out/trades.csv (6702 сделки)**, детекция/вселенная/капы НЕ
  перезапускаются (ноль нового симуляционного дрейфа).
- Шорт (primary): BTCUSDT, нотионал β× лонг-ноги.
- Горизонт: те же входы/выходы из trades.csv (24ч), БЕЗ подбора нового горизонта.
- Вес позиции 1/5, без компаундинга, атрибуция на дату выхода (учётная модель v2).

## 2. Бета (формула Codex, без пространства трактовок)
```
beta = cov(r_alt, r_BTC) / var(r_BTC)
окно: последние 30 календарных дней 30m closed returns до signal timestamp
      (бары со start ∈ [T+30м−30д; T−30м] — сигнальный бар ИСКЛЮЧЁН);
      r_t = close(t)/close(t−30м) − 1, только пары, где обе серии имеют оба бара
минимум: ≥480 общих точек (≈10 дней 30м-сетки), иначе — см. исключение
клип: β ∈ [0.2; 3.0] (одобренный risk cap; клипы считаются)
НЕТ беты / мало истории → сделка ИСКЛЮЧАЕТСЯ из primary sample
      (вето Codex: β=1-fallback протаскивает скрытую рыночную экспозицию
      молодых монет); исключённые — отдельный счётчик.
β=1-fallback существует ТОЛЬКО как diagnostic sensitivity (не вердикт).
```

## 3. Ноги, издержки, funding (обе ноги как отдельные позиции)
- long: 1× Radar-asset (вход/выход/цены — из замороженного trades.csv);
- short: **β × BTCUSDT**, вход одновременно по open того же входного 30м бара,
  выход по close того же выходного бара; нет BTC-бара на метке → исключение (счётчик);
- издержки: cost grid УБИТОГО теста **на ногу** {0.14 / 0.31 (база) / 0.71}%,
  итого сделки = c × (1 + β) — BTC-нога несёт СВОИ комиссии пропорционально нотионалу;
- funding: фактические события КАЖДОЙ ноги из её собственной funding-истории
  (BTC-funding ≠ funding альта); лонг платит +rate монеты, шорт BTC ПОЛУЧАЕТ +rate·β;
- Alpha-net(c) = (ret_coin − β·ret_btc) − c·(1+β) − f_coin + β·f_btc.

## 4. Окна и статистика (наследуются от six_year)
Sample 2021-10-17…2026-07-10 · train ≤2023 · val 2024 ·
**test_verdict 2025-01-01…2026-04-30** · слайс 2026-05+ = contaminated (вне вердикта).
Day-cluster t · block bootstrap 7д×10к · maxDD · Sharpe/Sortino.

## 5. Вердиктные ворота (все — иначе KILLED)
1) alpha-net > 0 при 0.62% на full И val И test_verdict;
2) выживает при 1.42% (знак test_verdict не флипается, full > −0.5·|full@0.62|);
3) ablations держат знак и t>1: без топ-5 символов, без топ-5 дней, возраст ≥180д;
4) test_verdict: alpha-net > 0 И day-cluster t ≥ 2.0.
Формулы вердикта: CONFIRMED FOR PAPER ONLY / INCONCLUSIVE / KILLED–NO-TRADE.
Потолок paper-only сохраняется (survivor-bias наследуется: срез мёртвых монет
ЗАВЫШАЕТ альфу лонг-ноги → результат = верхняя граница).

## 6. Вторичные срезы (НЕ вердиктные)
- β=1-fallback sensitivity (включая исключённых из primary) — диагностика вето;
- awakening vs single раздельно; β-бакеты (β<0.8 / 0.8–1.5 / >1.5);
- gross-альфа до костов (только для интерпретации исхода №2 из §8);
- диагностическая alt-корзина (equal-weight топ-20 PIT без сигнальной монеты) —
  ОТЛОЖЕНА, может быть посчитана после вердикта, вердикт не меняет.

## 7. Множественность
Тест №2 семейства на тех же данных (после six_year). Порог t≥2.0 на untouched
test + требование одновременного прохода трёх окон — заявленная компенсация;
дополнительных попыток на этих данных после вердикта НЕ предпринимается.

## 8. Предрегистрированные интерпретации исходов
1) альфа ≈ 0 → радар = ловец альт-беты; свечная ветка радара ИСЧЕРПАНА
   ОКОНЧАТЕЛЬНО (обязательство: не возвращаться к порогам/паттернам на свечах);
2) альфа > 0 но < костов → информация в отборе есть, монетизации через эту форму
   нет; дорога только к механизмам (OI/flows) или радикально иному исполнению;
3) альфа > костов на test_verdict → первый выживший; ТОЛЬКО paper-контур.
Априорная оценка (2026-07-10, до данных): исход 1-2 вероятнее — из six_year
выводится gross−funding ≈ −0.14%/сделку без хеджа; проход требует систематического
отставания BTC от сигнальной монеты >0.75%/24ч в окнах сигналов.

## 9. Лог поправок (до заморозки 20.07 — свободно; после — bugfix-only)

# ADVERSARIAL MEMO — оппонент собственной методологии (до полного прогона)

Вопрос: если у радара эджа НЕТ, способен ли текущий бэктест честно выдать KILLED —
или есть путь к искусственно красивому результату? PnL не смотрен. Scope не расширяю.

## Три опаснейших оставшихся источника ложноположительного результата

### 1. Параметры продакшена РОЖДЕНЫ ВНУТРИ test-окна (самый опасный)
**Механизм:** «замороженный» конфиг не спускался с неба: VOL_MULT 4→5 выбран
2026-07-02 по живому ретро июня-2026 («отбор лучших»), класс awakening ≥15× определён
2026-07-05 по кейсу VANRY (июнь-2026), капы/каскад настроены 02.07 по ретро
radar_hits. Всё это — данные ВНУТРИ untouched test (2025-01…2026-07). Формально
никакого подбора в бэктесте нет, но test содержит именно тот кусок истории, на
котором параметры отобраны. Классический «prereg поверх уже сделанного выбора».
**Направление:** завышает test-период (особенно 2026-05…07).
**Может ли сменить вердикт:** ДА — критерий §7.4 (test net>0, t≥2) может пройти
за счёт слайса, на котором конфиг калиброван.
**Минимальное исправление (внесено в PREREG §9 до прогона):** вердиктный
test-метрик считается на 2025-01-01…2026-04-30; слайс 2026-05-01…07-10
(окно генезиса параметров) печатается ОТДЕЛЬНО с меткой «contaminated by
parameter genesis» и в критерии §7.4 не участвует.

### 2. Оптимизм входа: первый принт после закрытия бара (латентный мираж)
**Механизм:** вход = open следующего 30м бара (t+0с после сигнала). Бой физически
не может: скан раз в 5 мин (медианная задержка ~2-3 мин) + доставка + реакция.
Если весь «эдж» живёт в первых минутах после спайка — бэктест его засчитает,
а живая система никогда не возьмёт. Прецедент у нас же: pump-диагноз
(MFE@2мин vs скан@5мин = структурный проигрыш) — та же ловушка.
**Направление:** завышает.
**Может ли сменить вердикт:** ДА — на стратегии класса «объёмный всплеск»
первые минуты часто и есть весь ход.
**Минимальное исправление (внесено в PREREG §9):** обязательная sensitivity
«запоздалый вход» — вход по CLOSE входного бара (+30м, консервативная граница
живой задержки). CONFIRMED требует сохранения знака primary на запоздалом входе;
флип знака ⇒ потолок INCONCLUSIVE. Primary-метрика не меняется — добавлен гейт.

### 3. Survivor-пул (известен, задокументирован — но всё ещё главный по величине)
**Механизм:** делистнутые исчезли из кандидатского пула целиком; спайки на
монетах, которые потом обнулились, в лонг-статистику не попадут.
**Направление:** завышает лонги (асимметрично).
**Может ли сменить вердикт:** способен превратить истинный KILLED в ложный
paper-CONFIRMED, если величина смещения велика.
**Исправление уже в предреге (достаточное как ограничение вывода):** потолок
вердикта CONFIRMED FOR PAPER ONLY + баннер UPPER BOUND + sensitivity ≥180д +
SHORT-зеркало асимметрии. Дальше бесплатно не чинится — только честная рамка.

## Независимость manual_check — оценка
✅ Доставка воспроизводится с нуля из сырых баров по frozen spec: пре-скан
кандидатов + глобальный хронологический fold (иная архитектура, чем у движка);
hitlog движка в определении доставки не используется; smoke_trades.json берётся
ТОЛЬКО как правая сторона диффа. Непустой дифф позиций = STOP (принято).
⚠ Остаточные пределы независимости (заявляю честно):
1) оба написаны одним автором — общее МИСЧТЕНИЕ спеки пройдёт оба (контрмера:
   спека и оба файла отревьюены внешней моделью — этим ревью);
2) MAJOR_SYMBOLS импортирован как версионированная данная (не логика);
3) оба едят одни и те же скачанные бары — дефект данных невидим обоим

## Решение
**A) SMOKE AUTHORIZED.** Дизайн способен честно убить гипотезу: вердиктные ворота
(0.71%-косты, ablations, day-cluster t, untouched test БЕЗ окна генезиса параметров,
